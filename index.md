# Heath Reinhard | Documentation Engineer

## AI Evals & Documentation Analytics

I started as a documentation engineer. I ended up building AI evaluation systems.

The pivot happened at Meta, where I was the first person to quantify how the company's internal AI assistants consume documentation — and discovered that AI consumption had already surpassed human consumption. That finding changed the documentation function's entire strategy. I then proved that AI answer quality is predictable from the characteristics of retrieved documents alone, which gave the team a scientific basis for deciding which docs to fix first.

From there I built the systems to act on those insights:
* a closed-loop quality flywheel that evaluates production AI agent trajectories, generates source-grounded fixes, and re-evaluates—driving enhanced outcomes for 40+ internal products.
* a documentation health platform that drove coverage from 44.1% to 73.9% across 600+ products.
* a 10-skill Claude Code plugin suite now standard tooling across the company.
* and a 5-agent AI fleet that lands 100+ code changes every two weeks.

Before Meta, I translated computing history for audiences from kindergarteners to professional engineers at the Living Computers museum in Seattle. Before that, I ran an IT services company for nine years.

I have a passion for brevity and an obsession with eliminating superfluous commas wherever I find them.
